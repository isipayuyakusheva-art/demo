# Универсальный сайт для демоэкзамена 09.02.07 (ГИА ДЭ БУ)

Готовый каркас веб-приложения, объединяющий требования **Модуля №1**
(проектирование и разработка ИС) и **Модуля №2** (дизайн веб-приложений).
Сделан так, чтобы за ~15 минут адаптироваться под **любую тему** (предметную
область), которая попадётся на экзамене.

Стек: **PHP (XAMPP) + MySQL + HTML + CSS + JS**.

---

## 1. Что уже реализовано

**Функционал (Модуль №1 и №2):**
- Регистрация пользователя (логин, пароль, ФИО, телефон, e-mail).
- **Уникальный логин** + серверная валидация всех полей (Модуль №2).
- Авторизация с **сообщениями об ошибке** ввода (Модуль №2).
- Личный кабинет: история своих записей + создание новой.
- Форма создания записи: выбор категории, дата/время, бинарный выбор (оплата),
  все поля обязательны.
- Панель администратора (`adminka` / `password`): все записи, **смена статуса**
  (в работе / выполнено / отменено) с **обязательной причиной отмены**.
- Адаптивный интерфейс + отдельные **мобильные макеты 390×844** (папка `mobile/`).

**Подсистема безопасности (критерий на 18 баллов в Модуле №1):**
- Хэширование паролей (`password_hash` / `password_verify`).
- PDO + подготовленные выражения → защита от **SQL-инъекций**.
- Экранирование вывода (`htmlspecialchars`) → защита от **XSS**.
- **CSRF-токены** на всех формах.
- Разграничение доступа по ролям (`user` / `admin`).
- Безопасная сессия (HttpOnly, SameSite) + `session_regenerate_id` при входе.

## 2. Запуск в XAMPP

1. Установите и запустите **XAMPP** → включите **Apache** и **MySQL**.
2. Скопируйте папку `universal-site` в `C:\xampp\htdocs\`.
3. Импортируйте базу: откройте `http://localhost/phpmyadmin` → вкладка
   **Импорт** → выберите файл `sql/schema.sql` → **Вперёд**.
   (или в консоли: `mysql -u root < sql/schema.sql`)
4. При необходимости поправьте доступ к БД в `config/app.php` (секция `db`).
5. Откройте `http://localhost/universal-site/`.

**Тестовые учётки:**
| Роль | Логин | Пароль |
|------|-------|--------|
| Администратор | `adminka` | `password` |
| Пользователь | `user` | `user1234` |

## 3. Структура проекта

```
universal-site/
├── config/
│   ├── app.php        ⭐ ВСЯ настройка темы (название, поля, цвет)
│   └── db.php            подключение к БД
├── includes/
│   ├── functions.php     безопасность, валидаторы, доступ к конфигу
│   ├── header.php        шапка
│   └── footer.php        подвал
├── assets/
│   ├── css/style.css     адаптивные стили
│   └── js/app.js         маска телефона, логика причины отмены
├── sql/
│   └── schema.sql     ⭐ таблицы + список категорий (сиды)
├── mobile/               макеты 390×844 (Модуль №2)
├── index.php             маршрутизация
├── register.php          регистрация
├── login.php / logout.php
├── dashboard.php         личный кабинет (история + новая)
├── create.php            форма создания записи
└── admin.php             панель администратора
```

## 4. Как адаптировать под новую тему

См. отдельный файл **`ADAPT.md`** — пошаговый чек-лист на 15 минут.
Коротко: правится только `config/app.php` (названия и подписи полей)
и список вариантов в `sql/schema.sql`. Код страниц трогать не нужно.
